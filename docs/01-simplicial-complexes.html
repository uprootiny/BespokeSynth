1. Simplicial Complexes

Building spaces from triangles, and understanding why it matters for sound.

1.1 What is a simplex?

Definition. A p-simplex is the convex hull of p+1 points in general position.
• 0-simplex: a point (vertex)
• 1-simplex: a line segment (edge)
• 2-simplex: a filled triangle (face)
• 3-simplex: a filled tetrahedron (but we won't need these)

We write a p-simplex as an ordered list of its vertices: σ = [v0, v1, ..., vp].

1.2 Simplicial complexes

Definition. A simplicial complex K is a collection of simplices such that:
1. Every face of a simplex in K is also in K.
2. The intersection of any two simplices in K is either empty or a face of both.

In plain English: if you have a triangle [0,1,2], you must also have the edges [0,1], [0,2], [1,2] and the vertices [0], [1], [2]. No dangling faces.

Example: Tetrahedron.
Vertices: {0, 1, 2, 3}
Edges: {[0,1], [0,2], [0,3], [1,2], [1,3], [2,3]} — 6 edges
Faces: {[0,1,2], [0,1,3], [0,2,3], [1,2,3]} — 4 faces

This is the boundary of a tetrahedron — a hollow triangular pyramid. It's a triangulation of the 2-sphere S².

1.3 The boundary operator

The boundary of a p-simplex is the alternating sum of its (p-1)-dimensional faces:

∂[v0, v1, ..., vp] = ∑_i (-1)^i [v0, ..., v̂_i, ..., vp]

where v̂i means "delete vertex i".

Example: boundary of an edge.
∂[0,1] = [1] - [0]

The boundary of an edge is its two endpoints, with a sign.
Example: boundary of a triangle.
∂[0,1,2] = [1,2] - [0,2] + [0,1]

The boundary of a triangle is its three edges, with alternating signs. If you traverse them in order, you go around the triangle.

The fundamental property: ∂∂ = 0

The boundary of a boundary is always zero. This is not obvious but it's the foundation of everything that follows. It means:

  • If you take the boundary of a triangle, you get a cycle of edges.
  • That cycle has no boundary of its own — it's closed.
  • This "closed things that aren't boundaries" idea IS homology.

1.4 Why this matters for sound

A simplicial complex is a space built from discrete pieces. Sound on this space is a function assigning a value (pressure, displacement) to each piece. The boundary operator tells you how these values relate to each other — how pressure at a vertex creates flow along edges, how flow along edges creates curl around faces.

The dual of the boundary operator is the coboundary operator δ. It goes the other direction: from vertices to edges, from edges to faces. This is exactly what we need for signal propagation.

DSP interpretation:
• Vertex values = node amplitudes (what you hear)
• Edge values = wave flows (how energy moves between nodes)
• Face values = cavity pressures (what resonates in enclosed regions)
• δ propagates signal upward through dimensions
• δ* propagates signal downward (feedback/reflection)

1.5 In our code

// Source/CohomologySynth.cpp

void CohomologySynth::AddFace(int v0, int v1, int v2)
{
   // Ensure vertices are sorted (canonical orientation)
   int sorted[3] = { v0, v1, v2 };
   std::sort(sorted, sorted + 3);

   // A face automatically requires its edges
   AddEdge(sorted[0], sorted[1]);
   AddEdge(sorted[1], sorted[2]);
   AddEdge(sorted[0], sorted[2]);

   mFaces[mNumFaces] = { sorted[0], sorted[1], sorted[2], mNumFaces };
   mNumFaces++;
}

void CohomologySynth::ComputeCoboundary0()
{
   // δ^0: C^0 → C^1
   // (δ^0 f)[i,j] = f(j) - f(i)  (gradient on edges)
   for (int e = 0; e < mNumEdges; ++e)
   {
      mDelta0[e][mEdges[e].v0] = -1.0f;  // -f(i)
      mDelta0[e][mEdges[e].v1] = +1.0f;  // +f(j)
   }
}
